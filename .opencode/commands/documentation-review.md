# Documentation Review Prompt

You are an expert technical writer. Perform a comprehensive documentation review of this codebase.

## Analysis Areas

### Code Documentation
- Review inline comment quality and coverage
- Analyze docstring completeness and clarity
- Check code example accuracy
- Review API documentation completeness

### Architecture Documentation
- Review system overview documentation
- Analyze component interaction diagrams
- Check deployment documentation
- Review configuration documentation

### User Documentation
- Review user guide completeness
- Analyze tutorial quality and accuracy
- Check FAQ coverage
- Review troubleshooting guides

### Developer Documentation
- Review setup and installation guides
- Analyze contributing guidelines
- Check coding standards documentation
- Review development workflow guides

### Maintenance Documentation
- Review changelog quality
- Analyze versioning documentation
- Check migration guides
- Review backup/recovery procedures

### Documentation Structure
- Review information architecture
- Analyze cross-references and linking
- Check searchability
- Review documentation versioning

## Output Format

Provide a detailed documentation assessment with:
1. CRITICAL: Missing documentation that blocks usage/development
2. HIGH: Significant documentation gaps affecting productivity
3. MEDIUM: Documentation improvements for better clarity
4. LOW: Minor documentation enhancements
5. GAPS: Specific missing documentation areas
6. RECOMMENDATIONS: Actionable documentation improvements

Focus on creating comprehensive, accurate, and accessible documentation.
