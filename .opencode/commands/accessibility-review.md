# Accessibility Review Prompt

You are an expert accessibility consultant. Perform a comprehensive accessibility review of this codebase.

## Analysis Areas

### Web Content Accessibility Guidelines (WCAG)
- Review WCAG 2.1 AA compliance
- Check semantic HTML usage
- Analyze keyboard navigation support
- Review screen reader compatibility

### User Interface Accessibility
- Check color contrast ratios
- Review focus management
- Analyze alternative text implementation
- Check form accessibility

### Assistive Technology Support
- Review ARIA labels and roles
- Check screen reader announcements
- Analyze keyboard shortcuts
- Review voice control compatibility

### Inclusive Design
- Check for cognitive accessibility
- Review motor disability considerations
- Analyze visual impairment support
- Check hearing impairment accommodations

### Mobile Accessibility
- Review touch target sizes
- Check gesture alternatives
- Analyze responsive design accessibility
- Review mobile screen reader support

## Output Format

Provide a detailed accessibility assessment with:
1. CRITICAL: Accessibility barriers that prevent user access
2. HIGH: Significant accessibility issues affecting usability
3. MEDIUM: Accessibility improvements for better inclusion
4. LOW: Minor accessibility enhancements
5. WCAG_COMPLIANCE: Specific WCAG guideline violations
6. RECOMMENDATIONS: Actionable accessibility improvements

Focus on creating an inclusive experience for all users regardless of abilities.
